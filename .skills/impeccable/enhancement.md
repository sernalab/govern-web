# Enhancement — animate, colorize, delight, onboard

Comandos para anadir personalidad y engagement. Usar con moderacion — menos es mas.

---

## animate

**Que hace:** Anade movimiento con proposito. No decorativo, funcional.

**Que hacer:**
- Transiciones de estado: entrada/salida de elementos, cambios de vista
- Feedback de interaccion: click, hover, drag
- Indicadores de progreso y loading
- Scroll-triggered reveals (sutil, no agresivo)
- Respetar `prefers-reduced-motion`

**Combina con:** `delight`

---

## colorize

**Que hace:** Usa color de forma estrategica para comunicar, no solo decorar.

**Que hacer:**
- Status indicators: exito (verde), error (rojo), warning (amarillo), info (azul)
- Diferenciacion de categorias o tipos
- Data visualization: graficos, badges, tags
- Enfasis en elementos interactivos vs informativos
- Mantener coherencia con la paleta del proyecto (consultar CLAUDE.md)

**Combina con:** `bolder`, `delight`

---

## delight

**Que hace:** Anade momentos de sorpresa positiva. Toques que hacen sonreir al usuario.

**Que hacer:**
- Micro-animaciones en momentos de exito (confetti, checkmarks animados)
- Easter eggs sutiles
- Copywriting con personalidad en momentos inesperados (404, empty states, loading)
- Transiciones elegantes entre estados
- Feedback haptico o visual satisfactorio

**Importante:** Delight es la sal — poca realza, mucha arruina. No forzar.

**Combina con:** `bolder`, `animate`

---

## onboard

**Que hace:** Guia al usuario nuevo. Reduce la friccion del primer uso.

**Que hacer:**
- Empty states informativos y con CTA claro
- Tooltips o hints en primera visita
- Progressive disclosure: no mostrar todo de golpe
- Defaults inteligentes que reduzcan decisiones iniciales
- Flujo de primer uso simplificado vs flujo experto

**Combina con:** `clarify`, `delight`
